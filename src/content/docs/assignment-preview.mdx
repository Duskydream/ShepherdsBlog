# 题解正文

# 一、排序、二分与分治（10题）

**算法本质：** 通过**分而治之**与**单调性**，将大规模问题降维打击。排序建立次序，二分利用次序在对数时间内逼近答案，分治则是将大问题拆分为结构相同的独立小问题。

**核心思想：**
1. **二分查找**：通过不断缩小区间，在 $O(\log N)$ 时间内找到目标，关键在于单调性的判定。
2. **分治法**：拆分（Divide） -> 解决子问题（Conquer） -> 合并结果（Combine），如归并排序的合并。

**避坑指南：** 二分边界极易写错（死循环或漏解），需明确区间开闭；计算中间值 `mid = l + (r - l) / 2` 防溢出。

## 1. <a href="http://xmuoj.com/contest/362/problem/LinK06" target="_blank">LinK06</a> 两数之和 <Badge text="双指针 / 有序数组夹逼" variant="note" size="medium" />

> **算法动机：** 数组预排序后元素具备单调性，相加和随首尾指针收缩呈单调递增/递减变化，可将双重循环优化为单次遍历。

**思路：** 在升序数组两端移动指针，依据当前和调整。

**关键代码：**

```cpp
// 提前对数组排序；保证双指针的移动有单调性。
int l=0,r=n-1; // n指代nums.size()，下同
while(l<r){ 
 int s=nums[l]+nums[r];
 if(s==target) 
     break;
 if(s<target) 
     ++l; 
 else --r; // 通过两数之和调整左/右指针
}
```

**复杂度分析：** 时间复杂度 $O(N \log N)$（含排序），空间复杂度 $O(1)$

**总结：** 本题重点是掌握双指针的基本建模与边界处理（比如**当指针移动时，要确保不会越过对方**，即循环条件严格为 `l < r` 的开区间；另外，若题目要求返回下标（原数组下标），则排序后需额外记录原索引，或使用哈希表处理）

**🔗 相关题目推荐：** <a href="http://xmuoj.com/contest/362/problem/LinK07" target="_blank">LinK07 三数之和</a>

## 2. <a href="http://xmuoj.com/contest/362/problem/LinK07" target="_blank">LinK07</a> 三数之和 <Badge text="排序 + 双指针 / 去重处理" variant="note" size="medium" />

> **算法动机：** 固定第一个数后转换成“两数之和”子问题，利用排序去重与双指针夹逼，避免暴力 $O(N^3)$ 搜索。

**思路：** 排序后固定一个数，再对剩余部分做双指针并去重。

**关键代码：**

```cpp
vector<vector<int>> res;// 用三元组存储答案
// ...    
sort(nums.begin(),nums.end());
for(int i=0;i<n;i++){ 
    if (i&&nums[i]==nums[i-1]) // 固定起点跳过相同值，开始枚举
        continue;
 for(int l=i+1,r=n-1;l<r;){ /*移动并去重，其中左右指针分别跳过相同值*/ } }
```

**复杂度分析：** 时间复杂度 $O(N^2)$，空间复杂度 $O(1)$

**总结：** 本题重点是掌握排序+双指针的基本建模与边界处理。

**🔗 相关题目推荐：** <a href="http://xmuoj.com/contest/362/problem/LinK06" target="_blank">LinK06 两数之和</a>

## 3. <a href="http://xmuoj.com/contest/362/problem/LinK27" target="_blank">LinK27</a> 大数排序 <Badge text="比较排序 / long long 数据范围" variant="note" size="medium" />

> **算法动机：** 数据范围突破常规整数界限，利用字符串长度与字典序比较定义严格弱序进行排序。

**思路：** 使用 O(nlogn) 的快速排序处理十万级数据。

**关键代码：**

```cpp
int i = l - 1, j = r + 1, x = q[l + r >> 1];
while (i < j) {
    do i++; while (q[i] < x);
    do j--; while (q[j] > x);
    if (i < j) swap(q[i], q[j]);
}
quick_sort(q, l, j); quick_sort(q, i, r);
```

**复杂度分析：** 时间复杂度 $O(N \log N \cdot L)$，空间复杂度 $O(N \cdot L)$

**总结：** 本题重点是掌握排序的双指针基本建模与边界处理，在日常做题时可以调用sort或range::sort方法。

**🔗 相关题目推荐：** <a href="http://xmuoj.com/contest/362/problem/LinK28" target="_blank">LinK28 快速选择第k个数</a>

## 4. <a href="http://xmuoj.com/contest/362/problem/LinK28" target="_blank">LinK28</a> 快速选择第 k 个数 <Badge text="快速选择 / 分区思想" variant="note" size="medium" />

> **算法动机：** 仅需获得第 $k$ 小的元素而非全量有序，基于 QuickSort 分区思想每次仅递归处理半边区间，降低期望复杂度。

**思路：** 快排划分后只递归包含第 k 小数的一侧。

**关键代码：**
```cpp
int sl = j - l + 1;
if (sl >= k) return quick_select(q, l, j, k);
return quick_select(q, j + 1, r, k - sl);
```

**复杂度分析：** 时间复杂度 期望 $O(N)$，最坏 $O(N^2)$，空间复杂度 递归栈 $O(\log N)$

**总结：** 本题重点是掌握快速选择的基本建模与边界处理：**分治 + 剪枝**—每次只递归可能包含答案的那一侧，另一侧直接舍弃。
实际中也可以维护大小为 k 的最大堆，或者直接调用c++的nth_element方法。


**🔗 相关题目推荐：** <a href="http://xmuoj.com/contest/362/problem/LinK30" target="_blank">LinK30 归并排序</a>

## 5. <a href="http://xmuoj.com/contest/362/problem/LinK30" target="_blank">LinK30</a> 归并排序 <Badge text="分治 / 有序区间合并" variant="note" size="medium" />

> **算法动机：** 标准的分治范式，将大数组递归二分为子区间各自排序后进行双指针合并，保证稳定的最坏时间复杂度。

**思路：** 递归分治并合并两个有序子区间。

**关键代码：**
```cpp
// 先递归，再归并，使用临时数组来存放归并的子区间
while(i<=mid&&j<=r) tmp[k++]=num[i]<=num[j]?num[i++]:num[j++]; //每次取较小的放入临时数组
while(i<=mid) tmp[k++]=num[i++];// 扫描左侧放入临时数组
while(j<=r) tmp[k++]=num[j++]; // 扫描右侧放入临时数组
```

**复杂度分析：** 时间复杂度 $O(N \log N)$，空间复杂度 $O(N)$

**总结：** 本题重点是掌握归并排序的基本建模与边界处理。

**🔗 相关题目推荐：** <a href="http://xmuoj.com/contest/362/problem/LinK31" target="_blank">LinK31 求排列的逆序数</a>

Tips:  **归并排序和快速排序比较：**

**1、处理顺序相反** ; 快排先分区再递归（先整理位置），归并先递归再合并（先处理子问题再汇总结果）

**2、时间复杂度稳定性不同** ; 快排平均O(n log n)但最坏O(n²)（有序数组退化），归并稳定O(n log n)无最坏情况

**3、空间占用差异巨大** ; 快排O(log n)原地交换内存友好，归并O(n)需要额外临时数组空间开销大

**4、稳定性不同** ; 快排不稳定（交换打乱相等元素原始顺序），归并稳定（合并时相等元素先取左半保持原序）

# 二、递归、回溯与状态搜索（15题）

**算法本质：** 穷举与状态空间树的遍历。**递归**是程序的实现形式，**回溯**是带“撤销”动作的深度优先搜索（DFS），用于寻找所有可行解或最优解。

**核心框架：**
1. **边界条件**：什么时候终止搜索并记录答案。
2. **选择列表**：当前状态下有哪些分支可以走。
3. **状态回溯**：做选择 -> 递归 -> 撤销选择，恢复现场。

**常见优化（剪枝）：**
- **可行性剪枝**：如果当前状态已经不满足条件，直接 return。
- **最优性剪枝**：如果当前代价已经大于已知的最优代价，停止搜索。

}

 ## KMP 拓展：

**核心思想：** 利用已匹配的前后缀信息，失配时不回溯主串只移动模式串，将匹配从O(n×m)优化到O(n+m)。

**next数组含义（最关键）：** `ne[i]`表示模式串前i个字符中最长相等前后缀的长度（不含自身）。如`p="ababc"`中`ne[4]=2`（"ab"="ab"），`ne[5]=0`。求next本质是模式串自匹配：用前缀去匹配后缀，`j`记录已匹配长度，比较`p[i]`和`p[j+1]`，匹配则`j++`并记录`ne[i]=j`，失配则`j=ne[j]`回退。匹配过程同理：主串指针`i`遍历不回溯，模式串指针`j`表示已匹配长度，比较`s[i]`和`p[j+1]`，完全匹配（`j==n`）后输出起始位置`i-n+1`，然后`j=ne[j]`继续寻找重叠匹配。

**边界处理：** 下标从1开始留s[0]和p[0]空位；ne[1]=0；while中加`j&&`防止访问ne[0]；数组开n+1和m+1；匹配成功后j回退到ne[j]允许重叠；若求所有匹配位置则不退出继续循环。

**高效原因：** 主串指针永不回退，模式串智能跳跃，每个字符最多比较两次。常见应用包括求所有匹配位置、最小循环节（`n-ne[n]`，当`n%(n-ne[n])==0`时）等。

## 36. <a href="http://xmuoj.com/contest/357/problem/JD140" target="_blank">JD140</a> 三界相克 <Badge text="带权并查集 / 模 3 关系维护" variant="note" size="medium" />

**思路：** 记录节点到根的关系模3，路径压缩时同步更新关系。带权并查集在普通并查集的基础上，为每个节点增加一个权值`d[x]`，表示节点x到其父节点的关系。对于食物链问题，定义权值模3的意义：0表示与根同类，1表示吃根，2表示被根吃。路径压缩时，节点到根的关系等于节点到原父节点的关系加上原父节点到根的关系之和再模3。合并时根据给定的关系类型计算新的权值。

**关键代码：**

```cpp

        int root = find(p[x]); // 先递归找到根节点（此时p[x]的d已更新）
        d[x] = (d[x] + d[p[x]]) % 3; // 更新x到根的距离：原距离 + 父到根的距离
        p[x] = root; // 路径压缩：直接指向根
    }
    return p[x]; // 返回根节点
}
```

**复杂度分析：** 时间复杂度 均摊 $O(\alpha(N))$，空间复杂度 $O(N)$

**总结：** 本题重点是掌握带权并查集的基本建模与边界处理：

```cpp
// 带权并查集模板
// 查找 + 路径压缩 + 权值更新
int find(int x) {
    if (p[x] != x) {
        int root = find(p[x]);          // 先递归找根
        d[x] += d[p[x]];                // 更新权值：x到根 = x到父 + 父到根
        p[x] = root;                    // 路径压缩
    }
    return p[x];
}

// 合并：将 x 和 y 合并，已知 x 到 y 的权值为 w
void unite(int x, int y, int w) {
    int rx = find(x), ry = find(y);
    if (rx != ry) {
        p[rx] = ry;
        d[rx] = w + d[y] - d[x];        // 计算 rx 到 ry 的权值
        // 推导：d[x] + d[rx] = w + d[y]  →  d[rx] = w + d[y] - d[x]
    }
}

// 查询 x 到 y 的权值关系（需在同一集合）
int query(int x, int y) {
    if (find(x) != find(y)) return -1;  // 不在同一集合，关系未知
    return d[x] - d[y];                  // x 到 y 的权值
}
```

# 五、动态规划（8题）

**动态规划（DP）的本质：** 通过**状态**表示子问题，从子问题出发（**自底向上**，与递归不同），用**状态转移方程**描述子问题之间的关系，利用**最优子结构**和**重叠子问题**，将大问题拆解为小问题，用空间换时间避免重复计算。

**核心公式：** 当前状态 = f(历史状态)，即 `dp[i] = f(dp[j])`，通过已解决的子问题推导出更大问题的解。

**两个特征：**

1. **最优子结构**：大问题的最优解包含子问题的最优解
2. **重叠子问题**：子问题被重复计算（用记忆化/递推解决）

> **算法动机：** 从顶部到底部的路径最大权值，当前格子的最大值仅依赖于上一层相邻格子的最大值，采用自底向上 DP 递推可避免边界处理。

**思路：** 自底向上，当前位置加上下一层两个可达值中的较大者。

**关键代码：**

```cpp
vector<int> dp = triangle[N - 1]; // 取最后一行
	for (int i = N - 2; i >= 0; i--) { // 自底向上做N-1次迭代
		for (int j = 0; j <= i; j++) {
			dp[j] = triangle[i][j] + max(dp[j], dp[j + 1]); 
		} // dp[j]表示从底部到当前位置(i,j)的最大路径和
	}
cout << dp[0] << endl;
```

**复杂度分析：** 时间复杂度 $O(N^2)$，空间复杂度 $O(N^2)$ 或 $O(N)$

**总结：** 本题重点是掌握线性DP的基本建模与边界处理。

**🔗 相关题目推荐：** <a href="http://xmuoj.com/contest/357/problem/JD162" target="_blank">JD162 雪道寻长</a>

## 47. <a href="http://xmuoj.com/contest/362/problem/LinK63" target="_blank">LinK63</a> 林克的01背包 <Badge text="背包 DP / 倒序容量转移" variant="note" size="medium" />

> **算法动机：** 经典 01 背包问题，每种物品仅能选 0 或 1 次，采用倒序容量遍历确保状态计算时不重复使用当前物品。

**思路：** 容量倒序转移，保证每件物品最多选择一次。

**关键代码：**
```cpp
for(int i=1;i<=n;i++){ cin>>v>>w;
	for(int j=V;j>=v;j--)
  		dp[j]=max(dp[j],dp[j-v]+w);  // 倒序
}
```

**复杂度分析：** 时间复杂度 $O(N \cdot V)$，空间复杂度 $O(V)$

**总结：** 本题重点是掌握01背包的基本建模与边界处理。

**🔗 相关题目推荐：** <a href="http://xmuoj.com/contest/357/problem/JD164" target="_blank">JD164 背包限重</a>

**为什么倒序？**

倒序保证了在考虑第 `i` 个物品时，`dp[j-v]` 还**没有**被第 `i` 个物品更新过，即来自**前 i-1 个物品**的状态。

## 48. <a href="http://xmuoj.com/contest/362/problem/LinK71" target="_blank">LinK71</a> 最长上升子序列 <Badge text="序列 DP / LIS" variant="note" size="medium" />

> **算法动机：** 求解子序列单调递增的最大长度，定义 $dp[i]$ 为以 $i$ 结尾的 LIS 长度，枚举前驱 $j < i$ 满足 $a[j] < a[i]$ 进行状态转移。

**思路：** 以每个位置结尾的最长上升子序列由此前更小元素转移。

**关键代码：**
```cpp
for(int i=0;i<n;i++){ 
    f[i]=1; // 包含自身
	for(int j=0;j<i;j++) 
        if(num[j]<num[i])
  			f[i]=max(f[i],f[j]+1); // f[i] 表示以num[i]结尾的最长上升子序列的长度。
	ans=max(ans,f[i]);
}
```

**复杂度分析：** 时间复杂度 $O(N^2)$，空间复杂度 $O(N)$

**总结：** 本题重点是掌握LIS的基本建模与边界处理。

## 49. <a href="http://xmuoj.com/contest/362/problem/LinK75" target="_blank">LinK75</a> 最长公共子序列 <Badge text="双序列 DP / LCS" variant="note" size="medium" />

> **算法动机：** 两个序列的最长公共子序列问题，状态 $dp[i][j]$ 表示 $S_1[1..i]$ 与 $S_2[1..j]$ 的 LCS 长度，根据字符是否相等分类转移。

**思路：** 按两个前缀定义状态，末字符相同则左上转移。

**关键代码：**

```cpp
for (int i = 1; i <= n; i++) {
    for (int j = 1; j <= m; j++) {
        if (num[i] == b[j]) 
            dp[i][j] = dp[i-1][j-1] + 1; // 匹配: 同时往前退
        else 
            dp[i][j] = max(dp[i-1][j], dp[i][j-1]);  // 不匹配： 取较大值
    } // dp[i][j] 表示序列num[n]的前i个字符和序列 b[m]的前j个字符的最长公共子序列
}
cout << dp[n][m] << '\n';
```

**复杂度分析：** 时间复杂度 $O(N \cdot M)$，空间复杂度 $O(N \cdot M)$

**总结：** 本题重点是掌握LCS的基本建模与边界处理。

**🔗 相关题目推荐：** <a href="http://xmuoj.com/contest/362/problem/LinK71" target="_blank">LinK71 最长上升子序列</a>

## 50. <a href="http://xmuoj.com/contest/357/problem/JD170" target="_blank">JD170</a> 万城巡游 <Badge text="状态压缩 DP / 旅行商问题" variant="note" size="medium" />

> **算法动机：** 旅行商问题（TSP），顶点数较小（$N \le 20$），利用二进制位图 `mask` 压缩已访问城池集合状态，结合 DP 避免阶乘级暴搜。

**思路：** mask 表示已访问集合，枚举下一个未访问城市转移。

**关键代码：**
```cpp
for (int mask = 1; mask < (1 << n); mask++) {
    for (int i = 0; i < n; i++) if (mask >> i & 1) {
        for (int j = 0; j < n; j++) if (!(mask >> j & 1)) {
            dp[mask | (1 << j)][j] = min(dp[mask | (1 << j)][j], dp[mask][i] + w[i][j]);
        }
    }
}
```

**复杂度分析：** 时间复杂度 $O(2^N \cdot N^2)$，空间复杂度 $O(2^N \cdot N)$

**总结：** 本题重点是掌握状态压缩DP的基本建模与边界处理。

**🔗 相关题目推荐：** <a href="http://xmuoj.com/contest/357/problem/JD163" target="_blank">JD163 背包叠甲</a>

**`dp[mask][i]`**：已经访问的城市集合为 `mask`，**当前所在城市为 i** 的最小路径代价。

**状态转移：**
当前状态：(mask, i)  代价 = dp[mask][i]
选择下一个城市 j（未访问）
转移至：(mask|`1<<j`, j)  新代价 = dp[mask][i] + w[i][j]
